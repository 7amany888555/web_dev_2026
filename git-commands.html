Essential Git Commands for Beginners

Git commands help you save project history, move between versions, and share code with GitHub.

A Practical Command Reference

This page introduces the Git commands beginners use most often. Each command includes an example, a short explanation, and a copy button so you can practice the command in your own project folder.

What is Git?

Version Control on Your Computer

Git is a version control system that tracks changes in your project files. It lets you save checkpoints, compare work over time, and return to earlier versions when needed.

Command: git --version

git --version

Shows the installed Git version and confirms that Git is available on your computer.

Basic Git Setup

Identify Your Commits

Before making commits, set your name and email. Git uses this information to label the work you save in the project history.

Command: git config user.name

git config --global user.name "Your Name"

Sets the author name that appears on your commits.

Command: git config user.email

git config --global user.email "you@example.com"

Sets the email address connected to your commits.

Creating and Cloning Repositories

Start or Download a Project

You can create a new Git repository in an existing folder, or clone a repository from GitHub to get a local copy on your computer.

Command: git init

git init

Creates a new Git repository in the current folder.

Command: git clone

git clone https://github.com/username/repository-name.git

Downloads a GitHub repository to your computer.

Checking Status

See What Changed

Checking status is one of the safest habits in Git. It tells you which files changed, which files are staged, and what still needs attention.

Command: git status

git status

Shows the current state of the working directory and staged files.

Adding Files

Stage Work for a Commit

Staging means choosing which changes should be included in the next commit. You can stage one file or all changed files.

Command: git add file

git add index.html

Stages one specific file for the next commit.

Command: git add all

git add .

Stages all changed files in the current project folder.

Committing Changes

Save a Checkpoint

A commit is a saved version of your staged work. Good commit messages explain what changed in a clear and specific way.

Command: git commit

git commit -m "Add navigation links"

Saves staged changes with a short message describing the update.

Viewing History

Review Past Commits

Git history helps you understand how a project changed over time. It is especially useful when you need to find when a change was made.

Command: git log

git log --oneline

Displays a short list of recent commits in the repository.

Working with Branches

Develop Without Disturbing Main

Branches let you work on a feature or fix separately from the main version of the project. This keeps experiments organized and safer.

Command: git branch

git branch

Lists the branches in the current repository.

Command: git checkout branch

git checkout -b new-feature

Creates a new branch and switches to it.

Pushing and Pulling

Sync with GitHub

Pushing sends your local commits to GitHub. Pulling brings the latest commits from GitHub down to your computer.

Command: git push

git push origin main

Uploads commits from your local main branch to GitHub.

Command: git pull

git pull origin main

Downloads and merges the latest changes from GitHub.

Common Git Workflow

A Simple Routine

A common beginner workflow is to check your files, stage the changes, commit them, and push the commit to GitHub. This keeps your local work and remote repository up to date.

Command: everyday Git workflow

git status
git add .
git commit -m "Describe your changes"
git push origin main

Checks your work, stages changed files, saves a commit, and uploads it to GitHub.
